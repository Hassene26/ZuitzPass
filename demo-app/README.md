# ZuitzPass demo app

A full-stack demo of the **unlinkable** flow: Alice registers, connects World ID, and joins Bob's
event whose only condition is "be a human" — achieving on-chain exactly what we drove by hand
(Part-A credential → Circuit-B redeem → Circuit-A eligibility → `EligibilityGate.consume`).

## Architecture

- **Frontend** (`frontend/`, Vite + React): Alice's flow (register → connect World ID → claim
  humanity → join event) and Bob's flow (create event → see attendees).
- **Backend** (`backend/`, Node + Express, runs in WSL alongside `nargo`/`bb`/`forge`):
  - RP signing for World ID v4 (reuses the `@worldcoin/idkit-server` flow).
  - The `VerifiedHumansTree` **inserter** (Part A) and a **relayer** for `redeem` / `consume`
    (so Alice's wallet is never linked).
  - **Server-side proving**: computes witnesses (Poseidon via `circomlibjs`, Merkle siblings via the
    SMT contracts' `getProof`), writes each circuit's `Prover.toml`, runs `nargo execute` +
    `bb prove --oracle_hash keccak`, and submits the tx.

> Proving is server-side **for now** (the backend sees the master secret). Moving proving into the
> browser (so the secret never leaves the device) is the planned privacy upgrade — the flow and
> contracts don't change.

## Build order (each slice runnable)

1. ✅ **Foundation** — config, chain, Poseidon self-test (PASS), Bob's create-event.
2. ✅ **Alice — Part A + redeem** — Circuit-B proving works server-side (verified: `/redeem`
   returns a valid proof).
3. ✅ **Alice — join event** — Circuit-A proving + consume endpoint.
4. ✅ **Frontend** — Alice/Bob React app (IDKit World ID + MetaMask). *Untested end-to-end in-app.*

## Run (WSL)

The backend is **keyless** — it proves + returns transaction calldata; the frontend signs
everything with **MetaMask** (connect the deployer account: it owns the StatementRegistry and is the
VerifiedHumansTree writer).

```bash
# 1. backend (needs nargo + bb on PATH)
cd demo-app/backend
cp .env.example .env      # set REPO_ROOT + the World ID app/rp/signing values; leave RELAYER_KEY blank
npm install
npm run dev               # :8787 — must print "[poseidon] self-test PASS"

# 2. frontend
cd ../frontend
npm install
npm run dev               # :5175 (proxies /api -> :8787)
# open http://localhost:5175, connect MetaMask (deployer acct on World Chain Sepolia)
```

Flow: **Bob** creates an event ("be a human"). **Alice** registers → World ID → insert credential →
redeem → join. Each step that touches chain pops a MetaMask confirmation; each proof is generated by
the backend. Bob's panel shows attendees as **nullifiers, never identities**.
